MSA 관련 토론회를 했다.

24 Jun 2020

Reading time ~1 minute

MSA 토론회

이번 토론회에서는 외부 초청으로 오신 분이 계셨습니다.

나이가 지긋하신 분이 오셨는데, 개발이야기를 함께 하기에 연세도 상당히 많이 보이셨습니다.

소개를 해주셨는데, NEXTREE 대표님이라고 해서 살짝 놀랐습니다.

개발 이야기를 하는 회사의 대표님는 처음 뵈었습니다.

우선 이야기 해주신 몇가지 이야기중 기억에 남는 것은 아래와 같습니다.

  1. 이벤트 기반으로 개발 해야 한다.
  2. 데이터(값)가 아닌 도메인 기준으로 설계 해야 한다.
  3. 빅뱅 보단 점진적으로 MSA로 변환하는 것을 추천 한다.
  4. 단일 트랜젝션을 사용하지 않으면 성능이 개선된다.
  5. MSA는 FRONTEND와 BACKEND가 쌍으로 구성된다.
  6. 동일한 레벨의 API 호출은 안된다.(FRONTEND를 통해서 호출 하거나, 아래 레벨의 API만 호출 할수 있다.)

그 외에도 많은 이야기를 해주셨는데, 유익한 시간이었습니다.

즐거운 시간이었습니다.
